Chapter 03

HTML 프리뷰와 에디터의 실제 작동 방식

Open Design의 HTML 에디터는 Monaco 같은 코드 에디터가 중심이 아닙니다. 생성된 HTML을 sandboxed iframe에 넣고, host가 주입한 bridge script와 postMessage로 선택/검사/수정 흐름을 만듭니다.

Artifact 스트리밍부터 파일 저장까지

1stream deltaCodex text delta가 Web으로 들어옵니다.
2createArtifactParser()<artifact ...> 시작/청크/종료 이벤트를 생성합니다.
3live artifact stateProjectView가 streaming HTML을 artifact.html state로 유지합니다.
4preview renderFileViewer 또는 synthetic preview가 buildSrcdoc()로 iframe 문서를 만듭니다.
5persistrun 종료 후 validateHtmlArtifact()와 manifest 생성 후 writeProjectTextFile()로 저장.
6file tab저장된 HTML은 프로젝트 파일 탭으로 열리고 이후 편집 대상이 됩니다.

Manual Edit Mode 상세

1. 편집 대상 발견

MANUAL_EDIT_DISCOVERY_SELECTOR는 main, nav, section, article, header, footer, div, h1, h2, h3, p, a, button, img, strong, span입니다. bridge는 이 중 실제 크기가 있고 source mapping 가능한 요소만 target으로 보냅니다.

source mapping 기준은 data-od-id 또는 host가 주입한 data-od-source-path입니다. LLM 산출물에 data-od-id가 많을수록 편집 정확도가 좋아집니다.

2. iframe ↔ host 메시지 
// iframe → host
{ type: 'od-edit-targets', targets: [...] }
{ type: 'od-edit-select', target: { id, kind, label, fields, attributes, styles, outerHtml } }

// host → iframe
{ type: 'od-edit-mode', enabled: true }

host는 ev.source === iframeRef.current.contentWindow를 검사해 해당 iframe에서 온 메시지만 받습니다.

3. 패널과 patch 종류

ManualEditPanel은 layer list, content/style/attributes/html/source 탭, undo/redo history를 제공합니다.

  • set-text: 자식 element가 없는 텍스트 노드만 안전하게 변경.
  • set-link: href와 label 변경. nested markup이면 label 변경 제한.
  • set-image: src/alt 변경.
  • set-style: inline style property를 camelCase → kebab-case로 적용.
  • set-attributes: 안전한 attribute명만 허용, protected data-od-*는 보호.
  • set-outer-html: replacement HTML은 root element 정확히 1개여야 함.
  • set-full-source: 전체 HTML source를 직접 교체.
4. 원본 HTML 수정

source-patches.ts는 browser DOMParser로 원본 HTML을 파싱하고, data-od-id, data-od-runtime-id, path id 순서로 element를 찾습니다. 수정 후 full document면 <!doctype html> + documentElement.outerHTML로 직렬화합니다.

5. 저장/undo/redo

FileViewer.applyManualEdit()는 patch 적용 결과를 writeProjectTextFile(projectId, file.name, result.source, { artifactManifest })로 저장합니다. 파일이 외부에서 바뀐 경우 최신 source를 다시 읽고 history를 비워 덮어쓰기를 방지합니다.

Inspect Mode와 Comment Mode

Inspect Mode

selection bridge로 고른 element에 대해 color/background/font size/weight/padding/radius/text align 등을 host panel에서 조절합니다. runtime preview에는 od:inspect-set 메시지를 보내고, 저장 시 host가 가진 override map을 <style data-od-inspect-overrides>로 원본 HTML에 삽입합니다.

Comment Mode

동일한 selection bridge를 사용하지만 목적은 style override가 아니라 comment attachment 생성입니다. 선택 대상의 selector, label, currentText, htmlHint가 다음 Codex turn의 commentAttachments로 들어갑니다.

보안/격리 포인트

iframe sandbox

프리뷰 iframe은 기본적으로 sandbox="allow-scripts"입니다. allow-same-origin을 주지 않아 generated HTML이 host origin storage에 접근하지 못하게 합니다.

postMessage 신뢰 경계

host는 iframe source 검사를 수행합니다. Inspect override 저장은 iframe이 보낸 CSS가 아니라 host state에서 직렬화합니다. generated artifact가 forged postMessage를 보내도 source 저장을 직접 조작하지 못하게 한 설계입니다.

HTML patch 제한

outerHTML 교체는 root element 1개만 허용하고, attribute name regex와 protected attributes로 최소한의 구조 보호를 합니다. 하지만 sanitizer가 아니므로 untrusted HTML execution은 iframe sandbox에 의존합니다.

artifact 저장 검증

<artifact> body가 완전한 HTML 문서가 아니면 저장을 거부합니다. 기존 파일 편집 후 prose summary를 artifact로 감싸는 LLM 실수를 막기 위한 guard입니다.

기획서 기반 시스템에 필요한 HTML 에디터 요구사항

주의: Open Design의 manual edit은 HTML-aware patcher지만 AST 수준의 lossless editor는 아닙니다. 포맷/doctype/attribute order 변화가 생길 수 있습니다. 복잡한 기획서 기반 코드 에이전트에 붙일 때는 "디자인 artifact HTML" 편집용으로 한정하고, 실제 앱 코드 편집은 Codex tool path로 분리하는 것이 안전합니다.
