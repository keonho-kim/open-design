Chapter 01

시스템 구조와 런타임 경계

Open Design은 monorepo 앱이지만 기능 경계는 명확합니다. Daemon이 로컬 리소스와 agent 실행을 소유하고, Web은 채팅/프리뷰/편집 UI를 소유합니다. 프로젝트 파일은 Daemon API를 통해 저장됩니다.

구성 요소

Web app

apps/web는 Next.js + React 클라이언트입니다. 핵심 화면은 ProjectView.tsx, 프리뷰는 FileViewer.tsx, 질문 폼은 QuestionForm.tsx가 담당합니다.

Web은 LLM을 직접 호출하지 않습니다. /api/runs를 만들고 SSE를 구독합니다.

Daemon

apps/daemon은 Express + SQLite 로컬 데몬입니다. agent detect, Codex spawn, skills/design systems scan, 프로젝트 파일 저장, artifact lint, static routes를 담당합니다.

주요 진입점은 server.ts와 chat-routes.ts입니다.

Codex adapter

codex.ts는 codex exec --json --skip-git-repo-check --sandbox workspace-write를 구성합니다. 프롬프트는 argv가 아니라 stdin으로 전달됩니다.

Codex output은 json-event-stream으로 파싱되고 json-event-stream.ts에서 UI 이벤트로 정규화됩니다.

Resource registries

skills.ts와 design-systems.ts가 파일 시스템을 스캔합니다. built-in/user root를 분리하고 user root가 우선합니다.

정적 API는 static-resource-routes.ts에 모여 있습니다.

런타임 시퀀스

Web사용자가 메시지를 보냄streamViaDaemon()이 전체 대화 transcript, projectId, skillId, designSystemId, model/reasoning을 POST /api/runs로 전달합니다.
Daemonrun 생성 및 SSE 준비createChatRunService()가 run id, 상태, 이벤트 버퍼, child process 핸들을 관리합니다. /api/runs/:id/events는 재접속 가능한 SSE입니다.
Daemonsystem prompt 조합composeDaemonSystemPrompt()가 active skill, active design system, craft, memory, project metadata를 읽어 composeSystemPrompt()에 넣습니다.
DaemonCodex CLI 실행buildArgs()로 args를 만든 뒤 spawn(). promptViaStdin이라 child stdin에 composed prompt를 씁니다.
Daemonstream 정규화Codex JSON line: thread.started, turn.started, item.started/completed, turn.completed를 status/text/tool/usage/error로 변환합니다.
Web채팅/프리뷰 반영text delta는 artifact parser를 통과합니다. <artifact>는 live preview로 보여주고 완료 시 project file로 저장합니다.

Codex만 남길 때 제거 가능한 범위

반드시 유지
  • codexAgentDef에 해당하는 runner args, stdin prompt delivery, JSON event parser.
  • skills/design-systems/resource scanner와 prompt composer.
  • project cwd, active skill staging, linked read-only dir, generated image dir allowlist.
  • artifact parser, question-form parser, HTML preview/editor runtime.
  • SSE run lifecycle와 cancel/inactivity watchdog.
축소 가능
  • Claude/Gemini/Qoder/OpenCode/Copilot/Pi/ACP adapter registry와 per-provider parser.
  • Anthropic/OpenAI API fallback provider UI. 단, OpenAI Codex CLI 설정 UI는 유지.
  • 비-Codex MCP 전달 형식. Codex의 MCP/skill 로딩은 버전별로 분리 검증 필요.
  • media provider path는 이미지 생성 기능을 쓰지 않는다면 후순위.
